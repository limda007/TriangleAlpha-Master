# 常见问题

## Q1: 被控端启动后在主控看不到节点？

**检查清单**：

1. 确认被控端目录下有 `主控IP.txt`，且内容为主控的 IP 地址
2. 确认主控的 UDP 监听端口（默认 8888）未被防火墙阻断
3. 确认两台机器网络可达（ping 测试）
4. 如使用 Tailscale/ZeroTier 等内网穿透，确认 IP 地址填写正确

## Q2: 节点显示「在线」但脚本不启动？

1. 确认被控端目录下存在 `SlaveClientConsole.exe` 文件（TestDemo 启动检查）
2. 检查 TestDemo 脚本是否正常
3. 尝试右键该节点 → 启动/重启脚本

## Q3: 运行状态显示「⚠缺Key」或「⚠Key不一致」？

主控已保存 API Key 但该节点缺少或 Key 值不同。正常情况下主控会自动下发修复。如未自动修复：

1. 前往 验证码 Tab → 确认 Key 正确
2. 点击「保存并下发」手动推送

## Q4: 验证码余额查询失败？

1. 确认 API Key 正确且已充值
2. 检查主控机器的网络连接
3. 余额服务可能临时不可用，稍后重试

## Q5: 账号导入后数量不对？

- 格式必须是 `账号----密码`（四个短横线分隔），不是逗号或空格
- 重复的账号会自动跳过（以用户名去重）
- 空行会被忽略

## Q6: 等级/金币数据不更新？

节点状态数据来源优先级：

1. 被控端 UDP 心跳上报（每 3 秒）
2. 脚本停止时数据会保留最后已知值，不会被零值覆盖

如果长时间不更新，检查被控端是否正常运行。

## Q7: 如何批量操作特定节点？

1. 在节点表格中 Ctrl+点击 或 Shift+点击多选节点
2. 然后点击操作面板的按钮，操作仅作用于选中节点
3. 未选中任何节点时，操作会作用于全部在线节点（需二次确认）

## Q8: 配置下发后节点没有生效？

1. 确认节点状态为「在线」
2. 配置通过 TCP 指令端口发送，确认端口未被占用
3. 尝试重启脚本使配置生效

## Q9: 分销平台对接如何使用？

1. 前往 设置 → 分销平台
2. 启用开关，填写 API 地址、用户名、密码、分组名
3. 启用后，已完成的账号会在 5 秒内自动上传到平台
4. 平台取号后，本地账号状态自动更新为「已取号」
5. 连续认证失败 3 次会自动暂停，请检查账号密码

## Q10: 超时监控是什么？

在大屏模式的账号池底部，勾选「超时监控」并设置分钟数。如果某个节点在指定时间内状态没有更新，主控会自动重启该节点的脚本。

适用场景：脚本卡死、游戏崩溃等异常情况的自动恢复。

## Q11: 如何备份账号数据？

账号数据存储在主控 exe 同目录的 `accounts.db` 文件中。直接复制该文件即可备份。

## Q12: 主控可以管理多少个节点？

理论上没有硬性限制。实测 50+ 节点稳定运行。节点表格使用增量更新和防抖刷新机制，不会因节点数量多而卡顿。
