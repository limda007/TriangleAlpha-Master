## 1. 快速诊断（一键脚本）

将 `Diagnose-TAEnv.ps1` 复制到目标机器运行（需管理员权限）：

```powershell
# 在目标机器上以管理员身份运行
powershell -ExecutionPolicy Bypass -File Diagnose-TAEnv.ps1
```

或远程 SSH 执行：

```bash
scp Diagnose-TAEnv.ps1 administrator@<IP>:'C:\Users\Administrator\Desktop\'
ssh administrator@<IP> "powershell -ExecutionPolicy Bypass -File 'C:\Users\Administrator\Desktop\Diagnose-TAEnv.ps1'"
```

> **注意（PS 5.1 兼容性）**：
> - 脚本文件必须为 **UTF-8 with BOM** 编码，否则中文输出乱码。
>   转码命令：`$f='...ps1'; $c=[IO.File]::ReadAllText($f,[Text.Encoding]::UTF8); [IO.File]::WriteAllText($f,$c,(New-Object Text.UTF8Encoding($true)))`
> - 脚本已兼容 PowerShell 5.1（无三元运算符 `? :`、无 `switch($true)` 多分支陷阱）。

## 2. UmiOCR 诊断

### 2.1 概述

UmiOCR (`OCR.exe`) 是基于 Qt 的离线 OCR 服务，用于文字识别（验证码、等级、金币等）。

### 2.2 依赖项

| 依赖 | 说明 | 最低要求 |
|------|------|----------|
| `OCR.exe` | UmiOCR 可执行文件 | 必须位于 `<安装目录>\OCR\OCR.exe` |
| `RapidOCR-json` | OCR 推理后端 | 随 UmiOCR 捆绑 |
| 桌面会话 | Qt GUI 要求 | 必须有交互式桌面（RDP/VNC），Session 0 无法启动 |
| Visual C++ 运行库 | Qt 依赖 | VC++ 2015-2022 (x64) |

### 2.3 单独诊断命令

```powershell
# 1. 检查 OCR.exe 是否存在
$base = "C:\Users\Administrator\Desktop\TriangleAlphaOOOOO"
Test-Path "$base\OCR\OCR.exe"

# 2. 检查 OCR 进程是否运行
Get-Process -Name "OCR" -ErrorAction SilentlyContinue | Select-Object ProcessName, Id, StartTime

# 3. 检查 RapidOCR-json 进程
Get-Process -Name "RapidOCR-json" -ErrorAction SilentlyContinue | Select-Object ProcessName, Id

# 4. 测试 OCR HTTP 接口（默认端口 1224）
try {
    $resp = Invoke-WebRequest -Uri "http://127.0.0.1:1224/api/ocr" -Method POST -Body '{"base64":"","options":{}}' -ContentType "application/json" -TimeoutSec 5
    Write-Host "[PASS] OCR HTTP 接口响应: $($resp.StatusCode)"
} catch {
    Write-Host "[FAIL] OCR HTTP 接口不可达: $($_.Exception.Message)"
}

# 5. 检查 VC++ 运行库
Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\VisualStudio\14.0\VC\Runtimes\X64" -ErrorAction SilentlyContinue | Select-Object Installed, Major, Minor
```

### 2.4 常见问题

| 症状 | 原因 | 解决 |
|------|------|------|
| OCR.exe 不存在 | 部署不完整 | 复制 `OCR/` 目录到安装目录 |
| 进程存在但接口不通 | Qt 初始化失败（Session 0） | 用 `schtasks /IT` 在交互式会话中启动 |
| `RapidOCR-json` 多余实例 | 多次启动累积 | `taskkill /F /IM RapidOCR-json.exe` 后重启 OCR |
| VC++ 运行库缺失 | 系统未安装 | 安装 [VC++ 2015-2022 x64](https://aka.ms/vs/17/release/vc_redist.x64.exe) |

## 3. dm.dll (大漠插件) 诊断

### 3.1 概述

`dm.dll` 是大漠综合插件的 COM 组件，通过 `RegisterDmSoftConsoleApp.exe` （DmHelper）封装为命名管道服务 `MyWpfDmHelperPipe`，提供键鼠模拟和图像识别能力。

### 3.2 启动链

```
Launcher.exe
  ├─ regsvr32 /s dm.dll          (COM 注册)
  ├─ RegisterDmSoftConsoleApp.exe (DmHelper 常驻服务)
  │   └─ 监听 MyWpfDmHelperPipe (单连接模式)
  └─ TestDemo.exe
      └─ DmClient → NamedPipeClientStream("MyWpfDmHelperPipe")
```

Runtime 的 `RuntimeBootstrap.LaunchDmHelper()` 也会尝试启动 DmHelper 并以 60s 间隔守护。

> **⚠️ 管道单连接模式**：DmHelper 的 NamedPipeServerStream 为单连接模式，一次只能被一个客户端连接。
> 当 TestDemo 已连接管道时，`Test-Path "\\.\pipe\MyWpfDmHelperPipe"` 返回 `False`，
> 外部也无法连接管道 —— 这是**正常行为**，不代表 DmHelper 故障。
> RuntimeBootstrap.cs 注释也明确指出："不做管道连接测试 — 会抢占 TestDemo 的管道连接导致键鼠中断"。

### 3.3 依赖项

| 依赖 | 说明 | 位置 |
|------|------|------|
| `dm.dll` | 大漠 COM 组件 | `<安装目录>\dm\libs\dm.dll` |
| `regsvr32` | COM 注册工具 | `C:\Windows\SysWOW64\regsvr32.exe` (32-bit) |
| `RegisterDmSoftConsoleApp.exe` | DmHelper 管道服务 | `<安装目录>\dm\RegisterDmSoftConsoleApp.exe` |
| .NET Framework 4.7.2+ | DmHelper 运行环境 | 系统自带 |

### 3.4 单独诊断命令

```powershell
$base = "C:\Users\Administrator\Desktop\TriangleAlphaOOOOO"

# 1. 检查 dm.dll 是否存在
Test-Path "$base\dm\libs\dm.dll"

# 2. 检查 COM 注册状态
# dm.dmsoft 注册后会在注册表中创建 CLSID
$clsid = (Get-ItemProperty "HKLM:\SOFTWARE\Classes\dm.dmsoft\CLSID" -ErrorAction SilentlyContinue).'(default)'
if ($clsid) {
    Write-Host "[PASS] dm.dmsoft COM 已注册, CLSID=$clsid"
} else {
    Write-Host "[FAIL] dm.dmsoft COM 未注册, 需要 regsvr32"
}

# 3. 手动注册 dm.dll
# 注意: 必须使用 SysWOW64 下的 32-bit regsvr32
C:\Windows\SysWOW64\regsvr32.exe /s "$base\dm\libs\dm.dll"

# 4. 检查 DmHelper 进程
Get-Process -Name "RegisterDmSoftConsoleApp" -ErrorAction SilentlyContinue | Select-Object ProcessName, Id, StartTime

# 5. 检查命名管道是否存在
Get-ChildItem "\\.\pipe\" | Where-Object { $_.Name -eq "MyWpfDmHelperPipe" }

# 6. 测试管道连接
Add-Type -AssemblyName System.Core
$pipe = New-Object System.IO.Pipes.NamedPipeClientStream(".", "MyWpfDmHelperPipe", [System.IO.Pipes.PipeDirection]::InOut)
try {
    $pipe.Connect(5000)
    Write-Host "[PASS] 管道连接成功"
    $pipe.Close()
} catch {
    Write-Host "[FAIL] 管道连接失败: $($_.Exception.Message)"
}

# 7. 测试 COM 对象创建（PowerShell 32-bit 环境）
# 注意: 大漠是 32-bit COM，64-bit PowerShell 无法直接创建
# 使用 32-bit PowerShell:
& "$env:SystemRoot\SysWOW64\WindowsPowerShell\v1.0\powershell.exe" -Command {
    try {
        $dm = New-Object -ComObject dm.dmsoft
        Write-Host "[PASS] dm.dmsoft COM 创建成功, Ver=$($dm.Ver())"
        [System.Runtime.InteropServices.Marshal]::ReleaseComObject($dm) | Out-Null
    } catch {
        Write-Host "[FAIL] dm.dmsoft COM 创建失败: $($_.Exception.Message)"
    }
}
```

### 3.5 常见问题

| 症状 | 原因 | 解决 |
|------|------|------|
| `dm.dmsoft COM 未注册` | 未运行 `regsvr32` 或权限不足 | 以管理员运行 `regsvr32 /s dm.dll` |
| DmHelper 未运行 | Launcher 未启动 / 路径不对 | 手动启动或检查 `dm/` 目录 |
| 管道连接超时 | DmHelper 崩溃 / COM 初始化失败 | 重启 DmHelper, 检查 dm.dll 注册 |
| 管道被占用 | 多实例 TestDemo | `taskkill /F /IM TestDemo.exe` 保留一个 |
| 管道不可见 (Test-Path=False) | TestDemo 占用中（单连接模式） | 正常行为，DmHelper 在运行即可 |
| `SendCommandAsync` Harmony hook 失败 | async 状态机兼容性 | 不影响正常功能，仅缺少异常保护 |
| 键鼠无反应 | DM 未绑定窗口 / 窗口不在前台 | 确保游戏窗口可见且不被遮挡 |

## 4. ONNX (DirectML GPU) 诊断

### 4.1 概述

ONNX Runtime 通过 DirectML 提供 GPU 加速推理，用于物体识别模型（如敌人检测）。若 GPU 不可用则自动降级到 CPU（速度慢约 5-10 倍）。

### 4.2 依赖项

| 依赖 | 说明 | 位置 |
|------|------|------|
| `onnxruntime.dll` | ONNX Runtime 原生库 | **Costura 嵌入 TestDemo**（磁盘上通常不存在独立文件） |
| `Microsoft.ML.OnnxRuntime.dll` | ONNX Runtime managed | Costura 嵌入 TestDemo |

> **onnxruntime.dll 部署说明**：该 DLL 通过 Costura.Fody 嵌入在 TestDemo.exe 中，运行时释放到内存加载。
> 诊断时不应仅检查磁盘文件，应同时检查 `diag.log` 中的 `onnxruntime.dll loaded` 记录确认运行时加载成功。
| DirectX 12 | DirectML 底层 | Windows 10 1903+ 自带 |
| DirectML.dll | DirectML 运行时 | Windows 附带或随 ONNX RT 分发 |
| 兼容 GPU 驱动 | 支持 DX12 的 GPU | NVIDIA/AMD/Intel 驱动需更新到支持 DX12 |

### 4.3 单独诊断命令

```powershell
$base = "C:\Users\Administrator\Desktop\TriangleAlphaOOOOO"

# 1. 检查 onnxruntime.dll 位置
$onnxPaths = @(
    "$base\onnxruntime.dll",
    "$base\runtimes\win-x64\native\onnxruntime.dll"
)
$found = $false
foreach ($p in $onnxPaths) {
    if (Test-Path $p) {
        $size = (Get-Item $p).Length
        Write-Host "[PASS] onnxruntime.dll 找到: $p ($([math]::Round($size/1MB, 1)) MB)"
        $found = $true
        break
    }
}
if (-not $found) { Write-Host "[FAIL] onnxruntime.dll 未找到" }

# 2. 测试 LoadLibrary（原生 DLL 是否可加载）
Add-Type -TypeDefinition @"
using System;
using System.Runtime.InteropServices;
public class NativeLoader {
    [DllImport("kernel32.dll", SetLastError=true, CharSet=CharSet.Unicode)]
    public static extern IntPtr LoadLibrary(string path);
    [DllImport("kernel32.dll")]
    public static extern bool FreeLibrary(IntPtr handle);
}
"@
foreach ($p in $onnxPaths) {
    if (Test-Path $p) {
        $h = [NativeLoader]::LoadLibrary($p)
        if ($h -ne [IntPtr]::Zero) {
            Write-Host "[PASS] onnxruntime.dll LoadLibrary 成功"
            [NativeLoader]::FreeLibrary($h) | Out-Null
        } else {
            $err = [System.Runtime.InteropServices.Marshal]::GetLastWin32Error()
            Write-Host "[FAIL] onnxruntime.dll LoadLibrary 失败 (Win32 err=$err)"
        }
        break
    }
}

# 3. 检查 DirectX 12 支持
$dxdiag = dxdiag /t "$env:TEMP\dxdiag.txt" 2>$null
Start-Sleep 5
if (Test-Path "$env:TEMP\dxdiag.txt") {
    $content = Get-Content "$env:TEMP\dxdiag.txt" -Raw
    if ($content -match "Feature Levels:\s*(.+)") {
        Write-Host "[INFO] DirectX Feature Levels: $($Matches[1].Trim())"
    }
    if ($content -match "Driver Model:\s*(.+)") {
        Write-Host "[INFO] Driver Model: $($Matches[1].Trim())"
    }
    if ($content -match "Card name:\s*(.+)") {
        Write-Host "[INFO] GPU: $($Matches[1].Trim())"
    }
    Remove-Item "$env:TEMP\dxdiag.txt" -Force -ErrorAction SilentlyContinue
}

# 4. 检查 GPU 信息
Get-CimInstance -ClassName Win32_VideoController | Select-Object Name, DriverVersion, VideoProcessor, AdapterRAM | Format-Table -AutoSize

# 5. 检查 DirectML.dll
$dmlPaths = @(
    "$env:SystemRoot\System32\DirectML.dll",
    "$base\DirectML.dll"
)
$dmlFound = $false
foreach ($p in $dmlPaths) {
    if (Test-Path $p) {
        Write-Host "[PASS] DirectML.dll 找到: $p"
        $dmlFound = $true
        break
    }
}
if (-not $dmlFound) { Write-Host "[WARN] DirectML.dll 未在常见位置找到" }

# 6. 查看 diag.log 中的 ONNX 预热结果
$diagLog = "C:\ProgramData\.ta\logs\diag.log"
if (Test-Path $diagLog) {
    Select-String -Path $diagLog -Pattern "ONNX|GPU|DirectML|onnxruntime" -Encoding UTF8 | 
        Select-Object -Last 5 | ForEach-Object { $_.Line }
}
```

### 4.4 GPU 兼容性

| GPU 类型 | DirectML 支持 | 备注 |
|----------|--------------|------|
| NVIDIA GTX 900+ | ✅ | 驱动需支持 DX12 |
| NVIDIA RTX 系列 | ✅ | 最佳支持 |
| AMD RX 400+ | ✅ | 驱动需支持 DX12 |
| Intel UHD 600+ | ✅ | 性能有限 |
| 虚拟 GPU (QEMU/VirtIO) | ❌ | 无真实 GPU 通道 |
| GPU 直通 (IOMMU/PCIe passthrough) | ✅ | PVE 需正确配置 |

### 4.5 常见问题

| 症状 | 原因 | 解决 |
|------|------|------|
| `onnxruntime.dll not found` | DLL 为 Costura 嵌入，磁盘上无文件 | 检查 `diag.log` 是否有 `onnxruntime.dll loaded`（有则正常） |
| `GPU (DirectML) unavailable` | 显卡驱动不支持 DX12 | 更新 GPU 驱动 |
| MessageBox 弹窗 "ONNX 预热失败" | onnxruntime.dll 缺失 | 部署 onnxruntime.dll |
| CPU 推理太慢 | GPU 不可用 | 升级 GPU 或接受 CPU 模式 |

## 5. 常见问题速查表

| 组件 | 状态检查命令 | 正常标志 |
|------|------------|----------|
| UmiOCR | `Get-Process OCR` | 进程存在 |
| UmiOCR 接口 | `curl http://127.0.0.1:1224/api/ocr` | 200 响应 |
| dm.dll COM | 注册表 `HKLM:\SOFTWARE\Classes\dm.dmsoft` | CLSID 存在 |
| DmHelper | `Get-Process RegisterDmSoftConsoleApp` | 进程存在 |
| DM 管道 | `dir \\.\pipe\MyWpfDmHelperPipe` | 管道存在（TestDemo 运行时不可见为正常） |
| ONNX Native | `diag.log` 搜索 `onnxruntime.dll loaded` | 出现此行（磁盘无文件为正常，Costura 嵌入） |
| ONNX GPU | `diag.log` 搜索 `GPU (DirectML) available` | 出现此行 |
| OpenCvSharp | `diag.log` 搜索 `Costura preload OK` | 出现此行 |

### 5.1 已知无害异常

| 异常 | 频率 | 影响 | 说明 |
|------|------|------|------|
| `OpenCvSharpException: Failed to find dll "OpenCvSharpExtern"` | 每次 TestDemo 启动 | ❌ 无 | OpenCvSharp 内部 `WindowsLibraryLoader` 搜索路径与 Costura 释放路径不同，但 `RuntimeBootstrap.PreloadCosturaNativeDlls()` 已通过 Win32 `LoadLibrary` 提前加载，P/Invoke 正常工作 |
| `SendCommandAsync hook fail: 不支持所指定的方法` | 每次 TestDemo 启动 | ❌ 无 | Harmony async 状态机 hook 兼容性问题，仅影响异常保护层，不影响正常键鼠操作 |
